# 로드맵

## 현재 상태

### 완료

- 공개 챌린지 목록 및 상세 흐름
- 로그인 기반 시도 흐름과 회원 소유 이력
- 세션 기반 인증과 `USER` / `ADMIN` 역할
- 관리자 챌린지 CRUD 및 활성 토글
- 관리자 모델 자산 운영 기능
- 런타임 MySQL과 분리된 H2 테스트 프로필

### 진행 중

- 인증 분리 이후 공개 UX와 관리자 UX 정리
- 운영 문서와 실행 가이드 정돈

## 다음 마일스톤

1. 인증과 라우트 가드가 붙은 상태에서 관리자 경험을 더 다듬기
2. 회원 프로필과 계정 관리 기능 보강
3. 챌린지 생성 및 분석 운영 흐름 강화
4. 더 많은 사용자 흐름에 검색, 필터, 편의 기능 추가
5. MediaPipe 기반 검증 및 운영 도구 정교화

## 전달 원칙

- 작은 수직 슬라이스 단위로 배포합니다.
- 관리자 기능은 REST/CRUD를 기본으로 시작합니다.
- 큰 UI 개편 전에 백엔드 계약을 먼저 안정화합니다.
- 계획보다 실제 구현 상태를 기준으로 문서를 유지합니다.
