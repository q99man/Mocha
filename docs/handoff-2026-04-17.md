# Handoff 2026-04-17

## 오늘 한 일

- 관리자 페이지에서 Pose Landmarker 모델을 삭제하고 다시 업로드할 수 있는 흐름을 붙였다.
- MediaPipe 브리지에서 잘못된 `MEDIAPIPE_BRIDGE_MODEL_PATH` 가 남아 있어도 기본 모델로 fallback 하도록 보강했다.
- 랜딩페이지 리뷰 섹션을 화이트 톤 기반으로 재설계하고, 2열 무한 캐러셀 구조로 바꿨다.
- 리뷰 섹션의 좌우 퍼짐을 줄이고 캐러셀 속도를 올렸다.

## 주요 변경

### 1. 관리자 모델 삭제 / 재등록

- 관리자 모델 이력 카드에 `모델 삭제` 버튼 추가
- 삭제 API 추가:
  - `DELETE /api/admin/model-assets/pose-landmarker/{id}`
- 활성 모델 삭제 시:
  - DB 자산 레코드 삭제
  - 보관 파일 삭제
  - 런타임 모델 파일 삭제
  - 남아 있는 최신 보관 모델이 있으면 자동 재활성화
- 관련 파일
  - `backend/src/main/java/com/motionchallenge/admin/controller/ModelAssetController.java`
  - `backend/src/main/java/com/motionchallenge/admin/repository/ModelAssetRepository.java`
  - `backend/src/main/java/com/motionchallenge/admin/service/ModelAssetService.java`
  - `frontend/src/pages/AdminModelAssetsPage.tsx`
  - `frontend/src/shared/api/adminApi.ts`

### 2. 브리지 모델 경로 안정화

- 브리지 실행 시 예전 예시값 `C:\path\to\pose_landmarker_lite.task` 같은 잘못된 환경변수가 남아 있어도 기본 모델로 fallback 하게 수정
- 브리지에서 모델을 못 찾을 때 resolved path를 더 분명히 남기도록 보강
- 관련 파일
  - `mediapipe-bridge/app/analysis.py`
  - `mediapipe-bridge/run-bridge.ps1`

### 3. 랜딩 리뷰 섹션 개편

- 리뷰 섹션 배경을 더 밝은 화이트 톤 밴드로 조정
- 카드만 밝은 상태가 아니라 섹션 전체가 밝게 보이도록 수정
- 리뷰 데이터를 2개 레일로 나눠서 위/아래 반대 방향 무한 캐러셀로 변경
- 리뷰 레일 폭을 제한해서 너무 좌우로 퍼져 보이지 않게 조정
- 캐러셀 속도 상향
- 모바일에서는 애니메이션을 끄고 가로 스크롤 형태로 fallback
- 관련 파일
  - `frontend/src/features/landing/LandingUseCaseSection.tsx`
  - `frontend/src/app/landing.css`
