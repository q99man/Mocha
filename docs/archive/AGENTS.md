# 에이전트 작업 원칙

이 문서는 과거 작업 세션에서 사용하던 에이전트 운영 원칙 보관본입니다.

## 당시 핵심 원칙

- Motion Challenge MVP를 작은 단위로 계속 전진시킨다.
- 기능은 모듈화하고 과도한 단일 파일 집중을 피한다.
- Redis는 보조 캐시 용도로만 사용한다.
- 관리자 및 운영 기능은 REST/CRUD를 기본형으로 삼는다.
- 작업 이후에는 구조 점검, 다음 작업 제안, 문서 갱신을 반복한다.

## 현재 기준

- 현재 기준 원칙은 보관 문서가 아니라 루트 `README.md`와 현재 코드 상태를 우선합니다.
