# UI DIRECTION

## 목표
Mocha의 프론트 UI를 DJMAX RESPECT V에서 느껴지는 리듬 게임 특유의 속도감, 퍼포먼스 감성, 레이어형 정보 구조를 참고해 재해석한다.

중요한 원칙은 "게임처럼 보이게"가 아니라 "리듬 게임의 시각 문법을 웹 MVP에 맞게 번역한다"이다.

## 참고 분석 요약
DJMAX 계열 UI에서 반복적으로 보이는 특징:
- 대형 키비주얼과 강한 발광 효과
- 중앙 메인 정보 + 좌우 보조 패널 구조
- 숫자, 상태, 난이도, 진행률 같은 기계적 정보의 강한 위계
- 선택된 항목을 크게 강조하고 주변 항목은 보조 레이어로 배치
- 결과 화면에서 성과 연출을 먼저 보여주고 세부 수치를 뒤에 정리
- 평면 카드보다 유리 패널, 라인 프레임, 네온 하이라이트, 오버레이 바 사용

## Mocha에 맞는 재해석 방향
- 제품 정체성은 "모션 챌린지 웹 서비스"로 유지한다.
- DJMAX 감성은 분위기, 레이아웃, 상태 표현에만 반영한다.
- 업로드, 에러, 폼 입력, 빈 상태는 웹 서비스답게 읽기 쉬워야 한다.
- 한국어 텍스트는 짧고 명확하게 유지한다.
- 실제 게임 UI 복제는 피하고, "퍼포먼스 중심 대시보드형 UI"로 옮긴다.

## 비주얼 키워드
- neon
- glass
- rhythm
- stage
- signal
- performance
- system overlay

## 색상 방향
- 기본 배경: 아주 짙은 남색 또는 청록 계열의 어두운 그라데이션
- 포인트 1: 시안/민트 계열 발광
- 포인트 2: 마젠타/핑크 계열 보조 발광
- 강조 텍스트: 밝은 화이트
- 보조 텍스트: 푸른 회색
- 위험/오류: 붉은 톤이지만 전체 톤과 어울리도록 채도를 약간 낮춤

## 타이포그래피 방향
- 제목은 넓고 선명한 디스플레이 계열 느낌
- 숫자, 점수, 시간, 상태는 모노 또는 기계적인 느낌을 강화
- 본문은 가독성이 우선
- 긴 설명은 줄이고, 짧은 상태 문구와 라벨 중심으로 전개

## 레이아웃 원칙
### 공통
- 화면 전체는 "무대"처럼 보이게 만든다.
- 배경과 패널은 분리된 레이어처럼 보여야 한다.
- 카드 그리드보다 "선택 화면"과 "상태 패널" 느낌을 우선한다.

### 홈
- 한눈에 분위기가 전달되는 메인 스테이지 섹션 필요
- 핵심 진입점은 2~3개만 강하게 제시
- 현재 가능한 기능과 준비 중인 기능을 분리해서 보여준다

### 챌린지 목록
- 일반 커머스 카드 목록보다 곡 선택 화면에 가깝게 설계
- 선택 후보 간 위계가 있어야 한다
- 난이도, 길이, 분석 준비 상태를 시각적으로 빠르게 읽게 만든다

### 챌린지 상세
- 대표 비주얼, 챌린지 메타, 준비 상태, CTA를 한 화면에서 읽게 한다
- "지금 도전 가능한지"가 가장 먼저 보여야 한다
- 보조 정보는 시스템 패널처럼 분리한다

### 도전 시작
- 카메라 권한, 세션 상태, 업로드 가능 여부를 체크리스트처럼 보여준다
- 현재 단계가 명확해야 한다
- placeholder도 임시 화면이 아니라 정식 준비 콘솔처럼 보여야 한다

### 결과
- 점수와 헤드라인을 가장 강하게 보여준다
- 세부 수치와 상태는 그 다음 레이어에 배치
- "다시 도전" CTA가 가장 강해야 한다

### 기록
- 단순 리스트보다 기록 로그/세션 아카이브처럼 정리
- 준비됨과 완료됨을 시각적으로 확실히 구분

## 컴포넌트 원칙
- 버튼: 둥근 pill만 쓰지 말고, primary는 발광하는 solid bar 느낌으로 강화
- secondary action: 얇은 outline/glass button 사용
- 패널: 내부 구획선을 둔 console card 스타일
- 배지: 상태 전용 tone을 분리
- 메트릭 카드: 숫자 강조형으로 정리
- 섹션 헤더: 번호, 코드, 짧은 소제목 조합을 고려

## 애니메이션 원칙
- 과한 애니메이션은 피한다
- 등장 시에는 짧은 fade + rise 정도만 사용
- hover는 glow, translate, border-light 중심
- 진행 단계는 line sweep 또는 pulse 정도로 제한

## 구현 우선순위
### 1단계
- 전역 디자인 토큰 정리
- AppLayout 리디자인
- 홈 화면 리디자인

### 2단계
- 챌린지 목록/카드 리디자인
- 챌린지 상세 리디자인

### 3단계
- 도전 시작 화면과 카메라 패널 리디자인
- 결과 화면 리디자인

### 4단계
- 기록 화면 리디자인
- 문구, 로딩, 빈 상태 세부 조정

## 기능과 디자인의 경계
지금 바로 고정해도 되는 영역:
- 전체 톤앤매너
- 공통 레이아웃
- 목록/상세/결과의 정보 위계
- 상태 배지와 시스템 패널

조금 늦게 고정하는 게 좋은 영역:
- 실제 녹화 중 HUD
- 실시간 카운트다운 연출
- 자세 피드백 시각화
- 프레임 단위 비교 그래프

## 실무 권장 진행 방식
1. 먼저 전역 스타일과 공통 레이아웃을 바꾼다.
2. 홈과 챌린지 목록/상세로 분위기를 고정한다.
3. 그 다음 시작/결과 화면에 시스템 패널 스타일을 확장한다.
4. 마지막에 문구와 미세 인터랙션을 다듬는다.

## 현재 판단
Mocha는 이미 API와 화면 흐름이 어느 정도 연결되어 있으므로, 지금 시점부터 프론트 디테일 작업을 시작하는 것이 효율적이다.

다만 카메라 내부 실시간 UX는 아직 placeholder 비중이 높으므로, 우선은 "준비 콘솔"과 "업로드 흐름" 중심으로 스타일을 입히는 것이 안전하다.
