[랜딩페이지 UI UX 기본 방향 3원칙]

1. 필요없는 건 다 제거하고 컴팩트한 디자인을 지향한다.

2. 레퍼런스에서 디자인을 추출하여 비슷하게 구현한다.

3. 목적에 따라 디자인한다.

[세부 사항]

1. 우리 홈페이지의 기본 UI는 모노톤으로 구성한다.
   1-1. 블랙&화이트를 기본으로 하며 필요시 상황에 맞는 컬러를 적용한다.
   1-2. 기본 텍스트는 Magarine으로 설정, 폰트는 크기는 과하지 않게 한다.
   1-3. 아이콘이나 장식은 따로 요청이 있지 않는 이상 넣지 않는다.
   1-4. 불필요한 문구, 상세한 설명은 제거한다.

2. 우리 홈페이지의 작업 방식은 레퍼런스에 철저하게 따라간다.
   2-1. 레퍼런스는 사용자가 이미지나 영상을 첨부하여 제공한다.
   2-2. 첨부한 이미지 및 영상을 분석해 다음 작업에 적용한다.
   2-3. 사용자가 만족한 결과가 나올때 까지 레퍼런스 구현을 반복한다.
   2-4. 다음 작업에 첨부한 자료가 없으면 사용자에게 자료없이 진행하는지 물어본다.

3. 우리 홈페이지는 사용자가 챌린지를 하도록 유도하는 것이 목표다.
   3-1. Feature(핵심 기능 소개)
   3-2. showcase(챌린지 갤러리)
   3-3. Use case(사용 사례 및 후기)
   3-4. CTA(Call to Action)
   3-5. Footer

[참고 사항]

1. 작업을 마치면 다음 작업에서 적합한 GPT모델과 이성 수준 추천한다.

2. 3번 목적에 맞춰 Mocha\public\background\폴더의 1~5번 이미지와 영상을 확인하고 분석하여 3-x번에 적용한다.
